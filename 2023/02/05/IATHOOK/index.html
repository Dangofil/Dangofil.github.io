IATHOOK

发表于2023-02-05|更新于2023-05-10
|阅读量:

简介

IATHOOK是通过修改可执行文件的IAT表来修改可执行文件在外部函数调用的时候所调用的函数地址

程序调用外部函数过程（动态链接）

  1. 主程序调用外部函数（以MessageBoxA举例）

  2. 主程序在文件状态时，导入表中会包含user32.dll中MessageBoxA函数的导入序号

  3. 主程序在内存状态时（运行状态），通过导入表中的导入序号，将user32.dll中的MessageBoxA函数地址装载到导入表中的IAT表，并间接调用

什么是IAT表

IAT表是导入表的一部分

在文件状态时，IAT表和INT表都指向导入函数名称和导入序号

在内存状态时，IAT表变为指向导入函数（存储导入函数地址）

查看导入表及IAT表

定位数据目录表（相对PE字符串的偏移），通过数据目录表的第二个成员来获得导入表的RVA和大小，转换成FOA来查看

数据目录表结构

转换方法：通过rva来定位导出表所在的节，FOA = 导入表rva-段rva+段foa

25bc-2000+1200=17bc

通过x32dbg调试程序，imagebase+rva就是导入表在内存状态的实际地址0x004025bc

导入表结构

如上所示，导入表是由5个DWORD成员构成的结构体，Name的RVA是0x0000274C，加上imagebase

就是0x0040274C

可以看到导入表依赖的模块名字KERNEL32.dll

通过导入表确定依赖的函数名称

由两个成员决定，第一个成员指向了一个INT 表.最后一个成员指向了一个 IAT表.

INT :: 导入名称表 Improt Name Table

IAT:: 导入地址表 Improt Address Table

如上图，在文件状态下，两张表是一样的，但是所在位置是不一样的名字也不一样.一个叫做 INT 一个叫

做IAT

结构体大小: 4个字节. 他是一个联合体.找最大的.

里面有4个成员.为当前的4个字节起了四个名字. 真正有用的是下面两个.

也就是说有的时候需要用第三个成员.有的时候需要用第四个成员

第四个成员是指向一个 IMAGE_IMPORT_BY_NAME的结构的RVA

无论使用那个表.都能找到依赖当前模块的函数.

  1. 通过OriginalFirstThunk来定位int表

  1. 如果找到的四个字节最高位为1，那么这四个字节就是函数的导出序号

  2. 如果最高位不是1,那么找的就是一个 RVA ,一个指向IMAGE_IMPROT_BY_NAME的结构

  3. 这里很明显，是第二种情况，那么0x0040273c指向的就是IMAGE_IMPROT_BY_NAME

  4. 可见指向的函数名称为LoadLibraryA，高位两个字节D7是函数在导出表中的导出索引，后面就是

    以零结尾的函数名称

总结来说: 不管是INT表还是 IAT表. 主要看其高位值,高位为1,那么去掉高位,就是函数的序号. 高位为0.指向一个结构.这个结构保存了函数的导出序号.以及函数名称.

在IMAGE_IMPROT_BY_NAME 结构中的 HINT 如果不是空,那么这个序号(索引) 就是导出表的函数地址表的索引. 我们可以直接拿着这个索引去导出表中获取函数地址.

确定函数地址

在pe被加载进内存的时候，IAT表不在指向函数的名称或者序号，而是保存函数的地址

可以看到，导出表的FirstThunk成员的值为0x00002000，加上imagebase就是0x00402000，我们去看所指向的内容

可以看到是0x76270A30

可以验证出，IAT表此时保存的就是函数地址了

IATHOOK原理

  1. 编写DLL注入目标程序，在DLL文件中获取MessageBox的原函数地址。
  2. 通过代码的方式找到DOS/NT/FILE/Optional头偏移地址。
  3. 通过DataDirectory[1]数组得到导入表的起始RVA 并与ImageBase基址相加得到VA。
  4. 循环遍历导入表中的IAT表，找到与MessageBox地址相同的4字节位置。
  5. 找到后通过VirtualProtect设置内存属性可读写，并将自己的函数地址写入到目标IAT表中。
  6. 没有找到的话直接pFirstThunk++循环遍历后面的4字节位置，直到找到为止。
